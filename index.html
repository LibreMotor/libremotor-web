Open-source vehicle software interoperability

Patch, validate, and manage compatible in-car apps.

Libremotor helps advanced users build reproducible APK compatibility profiles, validate activation for their own vehicle, and keep the process documented.

Activation

Vehicle eligibility is checked server-side using a VIN-derived identity so builds are tied to the owner who requested support.

Profiles

Choose practical profiles such as guest video, signed-in video, navigation, or phone projection. The Hub handles technical placement details.

Reproducible patches

Recipes declare supported source versions, applied patches, output metadata, and verification steps.

How to use Libremotor

  1. Install the Hub app on the head unit and accept the current Libremotor terms.
  2. Enter the vehicle VIN in the Hub. The activation service checks eligibility and returns a short-lived activation token.
  3. Choose the profile you want: video without sign-in, video with sign-in, navigation, or phone projection.
  4. Use the Libremotor Manager to import your own source APKs, apply the matching recipe, verify the output, and export the rebuilt APKs.
  5. Install only the generated APKs that match the selected profile and keep the Hub installed for activation, profile selection, and recovery.

Before you start

Libremotor modifies APKs for interoperability testing. That can affect accounts, warranties, stability, app behavior, and vehicle support. Read and accept the terms before using advanced patching features.

The software is open source and experimental. It is not affiliated with any vehicle manufacturer, Google, YouTube, Waze, ReVanced, or OpenAutoLink unless a specific repository states otherwise.
