LeapOS Android Entry-Point Probe

Open this page from a real HTTPS URL in Vivaldi and tap the buttons directly. Do not paste an intent: URL into the address bar: Chromium normally requires a user-initiated click and a target activity marked BROWSABLE. Run these tests only while the vehicle is parked.

Environment

Tap “Show browser capabilities”.

Android Settings — automatic resolution

These do not force a package. Android may choose the available Settings implementation.

Android Settings — force com.android.settings

This is the regular AOSP Settings package. These are the highest-value tests when it exists alongside the automotive UI.

Car Settings — force com.android.car.settings

AOSP Car Settings activities are usually exported but not marked BROWSABLE, so Chromium is likely to block these. They are included because an OEM-modified Vivaldi or Settings manifest may behave differently.

DocumentsUI / Files probes

The standard file-input buttons should invoke DocumentsUI in OPEN/PICK mode and do not download anything. The explicit launcher links are low-probability because stock Chromium requires BROWSABLE activities.

DocumentsUI launcher alias com.android.documentsui/.LauncherActivity DocumentsUI FilesActivity com.android.documentsui/.files.FilesActivity DocumentsUI VIEW_DOWNLOADS android.intent.action.VIEW_DOWNLOADS

Result notes

  • If any Settings page opens, first try its search icon, then Back/Up to reach the Settings home.
  • If only the file picker opens, note whether USB appears in its side drawer and whether long-press/overflow exposes another Files view.
  • If every intent reports that the link cannot be opened, the browser is probably enforcing BROWSABLE and LeapOS has removed the remaining browser-reachable Settings entry points.
